Agent Identity

What is the System Prompt?

Defines the agent's role, persona, and rules. It is sent before every conversation — the LLM always follows these instructions.

Example — Financial Analyst Agent
You are a senior financial analyst specializing in venture capital and private equity.

Your job is to:
- Analyze financial data, metrics, and market trends
- Provide clear, data-driven insights
- Always cite sources when referencing market data
- Present numbers in a structured format

Tone: professional, concise, and objective.
Never speculate without clearly labeling it as an estimate.
Example — Customer Support Agent
You are a helpful customer support agent for Acme Corp.
Answer questions about products, pricing, and policies.
Always be polite and empathetic.
If you don't know the answer, say "I'll escalate this to our team."
Defines the agent's role, tone, and constraints. Applied to every conversation. Click ? Help above for examples.
What is the User Prompt Template?

A template that wraps the user's input before it is sent to the LLM. Use {{input}} as the placeholder — it is replaced with whatever the user types or sends via API.

Leave blank to pass the user's input directly without any wrapping.

Example — Structured Analysis
Analyze the following and provide a structured report:

{{input}}

Your response must include:
1. Executive Summary (2–3 sentences)
2. Key Metrics & Data Points
3. Risks or Concerns
4. Recommended Next Steps
Example — API Data Processing
The following raw data was received from an external system:

{{input}}

Extract and return a JSON object with:
- summary: one sentence description
- sentiment: positive / neutral / negative
- action_required: true or false
- priority: high / medium / low
Wraps the user's message before sending to the LLM. Use {{input}} where the input should appear. Click ? Help above for examples.

ℹ️ How it works

1. Configure — Set the agent's name, instructions, input source, and output destination.

2. Get API key — An API key is auto-generated. Use it to send input via REST API.

3. Process — The agent uses your system prompt + tools to answer questions.

4. Output — Response streams to chat and/or is POSTed to your webhook URL as JSON.
